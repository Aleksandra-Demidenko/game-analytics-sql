#  Продуктовый аудит и анализ монетизации мобильной игры "Zombie Revolution" (SQL + Excel)

##  1. Бизнес-контекст и задача
**Цель проекта:** Провести комплексный аудит продуктовых, маркетинговых и финансовых метрик мобильной игры "Zombie Revolution" на основе логов базы данных. Собрать разрозненные показатели активности, качества сессий, эффективности реферальной программы и стабильности монетизации в единый аналитический фреймворк для руководства.

---

##  2. Структура и схема базы данных (Схема Skygame)

Для анализа использовались 5 взаимосвязанных таблиц логов и справочников:

### 1. Таблица `game_sessions` (Логи игровых сессий)
* `id_user` (INT) — Уникальный идентификатор игрока.
* `start_session` (TIMESTAMP) — Дата и время начала игровой сессии.
* `end_session` (TIMESTAMP) — Дата и время конца игровой сессии.

### 2. Таблица `users` (Данные профилей игроков)
* `id_user` (INT) — Уникальный идентификатор игрока.
* `reg_date` (DATE) — Дата регистрации пользователя в игре.
* `dev_type` (VARCHAR) — Тип устройства / Операционная система (iOS / Android).

### 3. Таблица `referral` (Данные маркетинговой реферальной программы)
* `id_user` — Уникальный идентификатор приглашающего пользователя.
* `ref_reg` — Флаг/маркер, показывающий, зарегистрировался ли приведенный друг в игре (1 — да, 0 — нет).

### 4. Таблица `log_prices` (Исторический лог изменений цен / SCD Type 2)
* `id_item` — Уникальный идентификатор игрового объекта (предмета).
* `price` — Стоимость предмета во временном диапазоне.
* `valid_from` — Дата старта действия указанной цены.
* `valid_to` — Дата окончания действия указанной цены.

### 5. Таблица `item_list` (Продуктовый справочник игровых предметов)
* `id_item` — Уникальный идентификатор игрового объекта.
* `name_item` — Понятное наименование игрового объекта (например, 'Crystal').
* `type` — Тип игрового объекта (категория монетизации).

### 6. Таблица `monetary` (Логи внутриигровых покупок)
* `id_user` (INT) — Уникальный идентификатор игрока.
* `dtime_pay` (TIMESTAMP) — Дата и время совершения транзакции.
* `id_item_buy` (INT) — Идентификатор купленного игрового объекта.
* `cnt_buy` (INT) — Количество купленных объектов в рамках одной транзакции.

##  3. Ключевые блоки анализа и бизнес-инсайты

### 🔹 Блок 1. Базовые метрики активности и удержания (`01_basic_metrics_and_engagement.sql` + Excel)
* **Продуктовые метрики:** Написаны SQL-запросы для расчета классических показателей активности: **DAU** (день), **WAU** (неделя) и **MAU** (месяц). 
* **Анализ удержания:** Данные импортированы в Excel, где на их основе рассчитаны коэффициенты липучести продукта (**Sticky Factors: SFW и SFM**), отражающие регулярность возврата аудитории в игру.
* **Качество сессий:** Проведен анализ длительности сессий с отсечением короткого «мусорного» трафика (<5 минут). Реализован поиск наиболее вовлеченных игроков (когорта регистрации 2022 года).
* **Эффективность реферальной программы:** Рассчитана общая конверсия инвайтов в реальные регистрации. С помощью фильтрации `HAVING` локализована группа ключевых юзеров (пользователи с >5 приглашениями и конверсией друзей от 50%).

### 🔹 Блок 2. Технический аудит чистоты данных (`01_basic_metrics_and_engagement.sql`)
* **Критический инсайт:** В ходе анализа сессий обнаружен масштабный технический баг логирования — у части записей отсутствовало поле времени завершения игры `end_session`. 
* **Локализация багов:** Анализ данных в разрезе типов устройств показал, что **проблема затронула исключительно пользователей iOS (доля дефектных сессий составила ~19%)**, тогда как на Android доля ошибок оказалась менее 1%. Технический отчет с локализацией бага передан команде разработки для исправления.

### 🔹 Блок 3. Динамика выручки, ценовая эластичность и "вирусность" (`02_revenue_dynamics_and_user_behavior.sql`)
* **Точность финансовой отчетности:** Рассчитан ежемесячный доход компании в разрезе категорий товаров (`type`). 
* **Эффект от повышения цен (Инсайт по Кристаллам):** Проанализировано влияние повышения стоимости Кристаллов с 1 января 2023 года. Исследование показало низкую ценовую эластичность премиального игрового товара: среднее количество покупаемых кристаллов за одну транзакцию (`AVG(cnt_buy)`) не упало, а **общая выручка по категории резко выросла**, что подтверждает коммерческую успешность реформы.
* **Качество рекламных кампаний:** Подтверждена гипотеза маркетинга о том, что таргетированная реклама в ноябре-декабре 2022 года привлекла более вовлеченных игроков (их среднее время сессии существенно выше, чем у остальных когорт).
* **Расчет виральности (K-factor):** Написан  SQL-скрипт с использованием CTE для расчета итогового **K-factor** продукта, "вирусность" и способность аудитории привлекать новых пользователей.
